# رفع خطای "Service Not Found" در XMPlus

## 📋 خلاصه مشکل

در لاگ اخیر، مشاهده شد که:

1. ✅ Invoice در XMPlus با وضعیت `Paid` و `serviceid: 15864` ثبت شده
2. ❌ اما `account/info` → `services: []` (خالی!)
3. ❌ و `service/info` با sid=15864 → `"Service not found"` (code 208)

این یعنی **فاکتور پرداخت شده اما سرویس در پنل وجود ندارد یا حذف شده**.

---

## 🔍 علت‌های احتمالی

### 1. سرویس حذف شده از پنل
- احتمالاً service با sid=15864 قبلاً ساخته شده بود اما بعداً از پنل XMPlus حذف شد
- این می‌تواند دستی (توسط ادمین) یا خودکار (مثلاً انقضای زمان) باشد

### 2. مشکل userid mismatch
- سرویس ممکن است متعلق به کاربر دیگری باشد
- یا email/password استفاده شده با userid مالک سرویس مطابقت ندارد

### 3. مشکل در API پنل XMPlus
- API به درستی کار نمی‌کند
- یا database sync issue وجود دارد
- یا نسخه XMPlus با Client API سازگار نیست

### 4. Invoice دستی به Paid تغییر یافته
- اگر فقط با ویرایش مستقیم database invoice را Paid کرده‌اید (بدون تکمیل مسیر پرداخت)
- XMPlus معمولاً در این حالت سرویس نمی‌سازد

---

## 🔧 راه‌حل اعمال شده

### تغییرات کد (commit: `46fef6e`)

1. **تشخیص خطای "Service not found"**:
   - وقتی `service/info` خطای "Service not found" می‌دهد، سیستم آن را log می‌کند
   - بعد از **5 بار تلاش ناموفق**، یک خطای واضح throw می‌شود

2. **پیام خطای بهتر**:
   - به جای polling بی‌نهایت، پیام واضحی نمایش داده می‌شود که شامل:
     * علت‌های احتمالی (حذف شدن، userid mismatch، API issue)
     * راه‌حل‌های عملی (ورود به پنل، چک کردن services، تماس با پشتیبانی)
     * اطلاعات لازم (panel URL, email, invoice ID, service ID)

3. **قابلیت سفارشی‌سازی پیام**:
   - دو پیام جدید به `BotMessageSeeder` اضافه شد:
     * `err_service_not_found`: خطای "سرویس در پنل یافت نشد"
     * `err_paid_without_service`: خطای عمومی "Paid بدون سرویس"
   - می‌توانید از پنل ادمین (BotMessages) این پیام‌ها را ویرایش کنید

4. بررسی کنید که آیا service با sid=15864 وجود دارد یا خیر
5. لیست invoices را چک کنید و status آنها را ببینید

## 📊 تحلیل لاگ شما

از لاگ ارسالی:

```json
{
  "invoice_id": "1KNTBTJKPOI7GJB6QXRS",
  "status": "Paid",
  "serviceid": "15864",
  "paid_date": "2026-04-06 22:20:09"
}
```

اما:

```json
{
  "services": [],
  "count": 0
}
```

و:

```json
{
  "status": "error",
  "code": 208,
  "message": "Service not found"
}
```

### نکات مهم:

1. **چندین invoice با serviceid مشابه**:
   - sid 15864, 15878, 15880 در لیست invoices دیده می‌شود
   - اما **هیچ‌کدام** در `account/info` وجود ندارند

2. **احتمالاً همه services حذف شده‌اند**:
   - این یک مشکل گسترده‌تر نشان می‌دهد
   - ممکن است bulk deletion رخ داده باشد
   - یا مشکلی در database sync وجود داشته باشد

3. **توصیه**:
   - حتماً با پشتیبانی XMPlus تماس بگیرید
   - از آنها بخواهید database را بررسی کنند
   - چک کنید که آیا services واقعاً حذف شده‌اند یا فقط API مشکل دارد

## 📝 سوالات متداول

### Q: چرا بعد از 5 بار؟
**A**: این یک balance است بین:
- دادن فرصت کافی برای تاخیرهای موقت API
- جلوگیری از polling بی‌نهایت که منابع سرور را مصرف می‌کند
- ارائه بازخورد سریع به کاربر

### Q: اگر سرویس بعداً ساخته شد چه می‌شود؟
**A**: کاربر می‌تواند دوباره دکمه "✅ تأیید و اجرا" را بزند. سیستم دوباره polling را شروع می‌کند.

### Q: آیا می‌توانم تعداد تلاش را تغییر دهم؟
**A**: بله، در `XmplusProvisioningService.php` خط ~1303، عدد `5` را تغییر دهید:

```php
if (($i + 1) >= 5) {  // به مثلاً 10 تغییر دهید
```

### Q: چگونه پیام خطا را سفارشی کنم؟
**A**: 
1. وارد پنل ادمین شوید
2. به بخش "Bot Messages" بروید
3. key `err_service_not_found` را پیدا کنید
4. content را ویرایش کنید
5. متغیرهای موجود: `{invoice_id}`, `{service_id}`, `{panel_url}`, `{email}`
