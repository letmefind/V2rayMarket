# 🔄 تمدید سرویس و پکیج‌های ترافیک

## ✅ وضعیت فعلی

### 1. تمدید سرویس (Service Renewal)

**✅ پیاده‌سازی شده و کار می‌کند**

#### نحوه کار:
1. کاربر روی "تمدید سرویس" کلیک می‌کند
2. سیستم `originalOrder->panel_client_id` (که همان `sid` است) را پیدا می‌کند
3. درخواست `POST /api/client/service/renew` با `sid` ارسال می‌شود:
```json
{
  "email": "tg4747@arashi.me",
  "passwd": "password",
  "sid": 15876
}
```
4. XMPlus یک invoice جدید برای تمدید همان سرویس ایجاد می‌کند
5. بعد از پرداخت، سرویس تمدید می‌شود (نه سرویس جدید!)

#### کد مربوطه:
```php
// XmplusProvisioningService.php - line 857
protected static function doRenewal(...) {
    $sid = $originalOrder->panel_client_id ?? null;
    if ($sid === null || $sid === '') {
        throw new RuntimeException('شناسه سرویس (sid) روی سفارش اصلی ذخیره نشده');
    }
    $sid = (int) $sid;
    
    $renew = $api->serviceRenew($email, $passwdPlain, $sid);
    // ...
}

// XmplusService.php - line 294
public function serviceRenew(string $email, string $passwd, int $sid): array
{
    return $this->postClient('/api/client/service/renew', [
        'email' => $email,
        'passwd' => $passwd,
        'sid' => $sid,
    ], 'service_renew');
}
```

#### نکات مهم:
- ✅ `sid` به درستی از `originalOrder->panel_client_id` گرفته می‌شود
- ✅ اگر `sid` وجود نداشته باشد، خطای واضح می‌دهد
- ✅ از API `service/renew` استفاده می‌کند (نه `invoice/create`)
- ✅ در ربات تلگرام کار می‌کند
- ✅ در وب هم کار می‌کند

## 🔧 تست تمدید سرویس

### مراحل تست:

1. **خرید اولیه**:
   - یک سرویس بخرید از ربات
   - بعد از پرداخت، سرویس فعال شود
   - در دیتابیس، `panel_client_id` (sid) ذخیره شود

2. **تمدید**:
   - وارد "سرویس‌های من" شوید
   - روی "تمدید سرویس" کلیک کنید
   - یک روش پرداخت انتخاب کنید
   - پرداخت را تکمیل کنید

3. **بررسی**:
   - سرویس باید **تمدید** شود (نه سرویس جدید)
   - در پنل XMPlus، همان `sid` باید expire date جدید داشته باشد
   - **نباید** سرویس جدیدی ساخته شود

### لاگ‌ها برای بررسی:

```bash
# بررسی لاگ XMPlus
tail -f /var/www/vpnmarket/storage/logs/xmplus-$(date +%Y-%m-%d).log | grep -i renew

# باید ببینید:
# - service_renew درخواست شده
# - sid ارسال شده
# - invid برای تمدید دریافت شده
```

### خطاهای احتمالی:

#### ❌ "شناسه سرویس (sid) روی سفارش اصلی ذخیره نشده"

**علت**: `originalOrder->panel_client_id` خالی است

**راه‌حل**:
1. سرویس را یک بار دیگر از پنل همگام کنید
2. یا سفارش جدید بگیرید
3. یا به صورت دستی در دیتابیس `panel_client_id` را set کنید:
```sql
UPDATE orders 
SET panel_client_id = 15876 
WHERE id = 123;
```

#### ❌ "تمدید ناموفق"

**علت**: API خطا برگرداند (ممکن است sid اشتباه باشد یا سرویس در پنل حذف شده)

**راه‌حل**:
1. لاگ را بررسی کنید
2. در پنل XMPlus چک کنید سرویس با همان sid وجود دارد
3. مطمئن شوید اطلاعات ورود کاربر درست است

## 📦 پیاده‌سازی پکیج‌های ترافیک (TODO)

### چیزهایی که باید اضافه شود:

#### 1. نمایش در ربات تلگرام

باید یک بخش جداگانه اضافه شود:
- دکمه "📦 خرید حجم اضافی" در منوی اصلی یا زیر سرویس
- لیست پکیج‌های ترافیک موجود
- انتخاب سرویس فعلی برای اضافه کردن حجم

#### 2. API خرید پکیج ترافیک

در XMPlus، احتمالاً API این شکلی است:
```
POST /api/client/invoice/create
{
  "email": "user@example.com",
  "passwd": "password",
  "pid": 123,              // شناسه پکیج ترافیک
  "billing": "topup_traffic",
  "sid": 15876             // سرویس فعلی که حجم به آن اضافه شود
}
```

#### 3. نحوه تشخیص پکیج ترافیک

پکیج‌های ترافیک معمولاً:
- در `billing` فقط `topup_traffic` دارند (نه `month`, `annual` و...)
- برای اضافه کردن حجم به سرویس موجود هستند
- قیمت ثابت دارند (بر اساس مقدار حجم)

#### 4. فلوی پیشنهادی:

```
کاربر: سرویس‌های من → انتخاب سرویس → 📦 خرید حجم اضافی
                                           ↓
                              لیست پکیج‌های ترافیک
                                           ↓
                              انتخاب پکیج (مثلاً +5GB)
                                           ↓
                              انتخاب روش پرداخت
                                           ↓
                              پرداخت و اضافه شدن حجم
```
